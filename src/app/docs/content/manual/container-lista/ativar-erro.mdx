Em alguns casos, a clonagem de componentes pode falhar, deixando o container com status de **Erro**. Isso pode ocorrer devido a fatores externos do ambiente ou instabilidades no produto.

Antes de prosseguir com a ativação, é recomendado analisar as mensagens de erro e tentar corrigi-las. No entanto, caso deseje seguir com a ativação do container, utilize o botão <SkipForward className="inline w-4 h-4 mx-1 text-muted-foreground" /> disponível na coluna **Container Actions**.

Esse botão só estará visível para containers com status de erro.

Ao ativar o container:
- Ele será habilitado, mesmo que haja componentes com erro.
- Os componentes com erro **serão ignorados nos testes**.
- Componentes originais (sem erro) serão utilizados durante a execução.

Essa funcionalidade permite dar continuidade ao processo de testes sem depender da correção imediata de todos os componentes com falha.
