### Opções de execução (Run Programs On)

- **Mainframe** (padrão): programas executam apenas no mainframe.
- **Distributed**: executam apenas em plataforma distribuída (requer a feature **ECCOX APT® EDGE APPLICATIONS FOR DDF**).
- **Mainframe and Distributed**: executam em ambos os ambientes.

### Selecionando componentes

A seleção dos componentes é feita via checkbox:

  <MdxImage
    src="/images/docs/test-container/components/selecionar-componentes/selecionar_componentes.png"
    alt="Seleção automática de componentes."
    width={1200}
    height={700}
    className="rounded-lg border"
  />
  <p className="mt-2 text-sm text-muted-foreground">Figura 37: Seleção automática de componentes.</p>
</div>

Após selecionar os componentes desejados, clique em **CONCLUDE** para iniciar a **validação de versões**. Se existirem componentes iguais já presentes no container com origens diferentes, será exibido o alerta de divergência.

### Lidando com divergências

<div className="my-6">
  <MdxImage
    src="/images/docs/test-container/components/selecionar-componentes/browse_divergences_origens.png"
    alt="Browse Divergences - Origens."
    width={1200}

Componentes em divergência serão destacados na coluna **Status Add** com mensagens como:

- No selected components were added because they are already in the container
- All selected components are with origin choice PENDING
- Some selected components were added, and others are already in the container
- Some selected components were added, and others are with origin choice PENDING

Para resolver:

<div className="my-6">
  <MdxImage
    src="/images/docs/test-container/components/selecionar-componentes/browse_divergences.png"
    alt="Browse Divergences."
    width={1200}
    height={700}
    className="rounded-lg border"
  />
  <p className="mt-2 text-sm text-muted-foreground">Figura 39: Browse Divergência.</p>
</div>

Escolha entre as opções:
- **Container**: mantém a versão atual.
- **Discovery**: substitui pela versão descoberta (com action `replace` se necessário).

Você pode usar os botões **ALL CONTAINER** ou **ALL DISCOVERY** para aplicar em lote.

### Finalizando o processo

Após resolver as divergências, finalize com **CONCLUDE**. Os componentes serão exibidos na aba **Container Preparation** da tela principal do container:

<div className="my-6">
  <MdxImage
    src="/images/docs/test-container/components/selecionar-componentes/componentes_adicionados.png"
    alt="Componentes recentemente adicionados."
    width={1200}
    height={700}
    className="rounded-lg border"
  />
  <p className="mt-2 text-sm text-muted-foreground">Figura 40: Componentes recentemente adicionados.</p>
</div>

Use o botão **Execute Component Pending Action** para:
- Criar a primeira versão de clones;
- Atualizar clones existentes.

<MdxCallout type="warning"  title="Atenção">
 Os botões disponíveis podem variar conforme as features contratadas
</MdxCallout>
