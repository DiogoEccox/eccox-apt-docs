### Informações necessárias para criação

Ao criar um novo container, forneça os seguintes dados:

- **Test Container Name:** Descrição do container de teste.
- **Environment:** Ambiente principal para instalação do container.
- **Application:** Sistema ou aplicação ao qual o container pertence.
- **Test Plan:** Plano de teste vinculado ao container. A lista de planos disponíveis depende do Environment e Application selecionados. Para mais detalhes sobre a criação de planos de teste, consulte <Link href="/docs/manual/container-criacao/plano" className="text-blue-600 underline">Criar plano de teste</Link>.

Após preencher as informações, clique no botão **CREATE NEW CONTAINER** para finalizar a criação e ser redirecionado para o novo container.

### Acessando um container criado

Os containers podem ser acessados a qualquer momento através da função **Browse**, disponível na opção **Test Container** do menu lateral. Dentro da lista de containers, utilize os botões de:

- **Visualização ():** Para consulta dos detalhes do container.
- **Edição ():** Para realizar alterações no container.

Para mais informações sobre a função Browse, consulte a seção **Lista de containers**

### Criação de um novo plano de teste

Para criar um novo plano de teste, as opções **Test Container Name** e **Environment** precisam estar preenchidas. Em seguida, clique no botão **+** mostrado na Figura 4 para exibir a pop-up da Figura 7.

Após preencher os campos necessários, confirme a criação do plano de teste e ele será adicionado automaticamente à lista disponível para o container.
